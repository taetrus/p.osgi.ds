02Root causes

Four ways one panel freezes all

All four were in play — and the bottom three can't be fixed by moving services alone. They need the process split.

EDT

Blocking work

A slow call on the shared event thread freezes every panel at once.

EDT

Heavy render

A panel's own painting/compute hogs the one UI thread shared by all.

HEAP

GC pressure

One bundle's leak triggers stop-the-world pauses across the whole heap.

PROCESS

Crashes

An exception or deadlock corrupts shared state and takes the app with it.

05Target architecture

Desktop micro-frontends

One JVM per panel-group, each in its own window. The only wire between them is a shared service contract on the ECF bus.

08Final delivery

One window. Two processes.

The last ask: make it look like one UI showing both panels. A Swing widget can't cross processes — so you pick two of three, and dock two borderless windows to fake the third.

WISH 1

One window

A single combined UI on screen.

WISH 2

Real panels

Actual interactive widgets, as-is.

WISH 3

Isolated processes

Separate EDT · heap · crash domain.

Native embedding gives all three — but only on Linux/Windows, not macOS. So we keep 2+3 (real, isolated) and approximate 1: an anchor window (App-1) publishes its live screen bounds over ECF; a follower (App-2) glues itself to the right edge. Two windows wearing one skin.

09The grid

From two windows to a tiled grid

The dock proved isolation with one window each. The next step generalised it: each app opens N windows, and both apps tile into one shared grid — master frames in teal, detail in amber, interleaved and wrapping. Same @Reference story, now multiplied.

SLOT MATH

Even ↔ odd interleave

masterSlot(k)=2k
detailSlot(k)=2k+1

One global slot sequence; master takes the evens, detail the odds. Fill order alternates M, D, M, D…

COORDINATION

One DockLayout

The master computes the grid once and publishes it over ECF. The detail reads it back and runs the identical slotBounds(). Shared parameters, not shared state — no per-frame messaging.

PANELS

Split per frame

distribute() spreads each app's panels across its frames as balanced contiguous chunks. N = 1 collapses back to chapter 08's single window — no regression.

10Takeaways

What carried the day

A

Question the means, not the requirement. "Transfer a panel" was an assumption; the goal was isolation.

B

One constraint did all the damage. Insisting on a single window forced every expensive path. Dropping it made it cheap.

C

Isolation is free; coordination is cheap. Separate processes give it for free; ECF services wire them back up.

D

Spike to find the gap. A few hundred lines proved the thesis — and exposed the discovery requirement early.

E

Generalise once it's proven. The dock became an N-frame grid by parameterising one layout — shared numbers, not new coupling.
